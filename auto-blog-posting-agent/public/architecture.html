Auto Blog Posting Agent

AI-powered blog generation pipeline that crawls your website, researches topics, writes SEO-optimized content, generates images, and auto-publishes on a schedule.

High-Level Architecture

Express API Server

The system has 4 layers: API Layer (Express), Orchestration Layer (LangGraph), AI Layer (Gemini), and Persistence Layer (file-based JSON storage + vector embeddings).

How RAG (Retrieval-Augmented Generation) Works

The RAG system ensures each new blog is unique and builds on previous content instead of repeating topics.

Site Registration
Crawl website → extract niche, features, audience
→
Embed & Store
Gemini converts text → 768-dim number vector → saved to vectors.json
→
Knowledge Base
Site overview + each blog's summary + title + content

When generating a new blog:

Topic Research uses RAG
Step 1: Coverage Map
Reads all stored entries to see "what topics have we already covered?"
Step 2: Similarity Search
Embeds the candidate topic → compares cosine similarity against all stored vectors → rejects topics > 82% similar to existing posts
Content Writing uses RAG
Context Injection
Top 5 most relevant past posts are injected into the AI prompt: "Here's what you already wrote. Don't repeat — go DEEPER."
Post-Save Indexing
After each blog is saved, its summary + title are embedded and added to the knowledge base so the NEXT blog can see it.

RAG prevents this problem:

Without RAG
Blog #1: "Invoicing Basics"
Blog #2: "Invoicing Basics for Beginners" ← repeats!
Blog #3: "Getting Started with Invoicing" ← repeats!
With RAG
Blog #1: "Invoicing Basics"
Blog #2: "Multi-Currency Payment Processing" ← unique!
Blog #3: "Tax Compliance for International Billing" ← unique!

Two-Mode Topic Discovery

Mode 1: Deepen (default)

Searches vector store for SIMILAR posts to the site's niche.

Strategy: "Write a follow-up, part 2, advanced version, or how-to guide based on what's already performing well."
Mode 2: White Space (exploreDifferent=true)

Looks at the full coverage map to find gaps.

Strategy: "Find topics in the niche that have ZERO coverage. Blind spots. Sub-niches. Contrarian angles."

Agent Details

Estimation Agent

Tools: search_knowledge_base, build_estimate

Flow:

  1. Receives customer requirements from Supervisor
  2. Calls search_knowledge_base with keywords from the message
  3. Finds matching services from pricing.json + uploaded documents
  4. Calls build_estimate to generate structured quote
  5. Returns: service list, min/max prices, timeline, policies, included features

Negotiation Agent

Tools: None (pure LLM reasoning)

Strategy:

  1. Phased Delivery — "Start with MVP at $X, add features in Phase 2"
  2. Reduced Scope — "Remove feature Y to bring cost down to $Z"
  3. Payment Plan — "Split into 3 monthly installments"

Booking Agent

Tools: get_calendar_slots, create_booking

Flow:

  1. Fetches next 5 available slots from Google Calendar (respects working hours + buffer time)
  2. Shows clickable slot cards to customer
  3. Collects name + email
  4. Creates Google Calendar event with Google Meet link
  5. Sends confirmation email via Nodemailer
